# 🎮 애니메이션 기반 2048 웹 게임 및 관리자 시스템

### 📝 주제 설명
본 프로젝트는 클래식 퍼즐 게임인 '2048'을 웹 브라우저에서 즐길 수 있도록 구현한 유저 친화적 웹 서비스입니다. 기존의 텍스트 기반 데이터 갱신 방식에서 벗어나 CSS Transition 좌표 계산을 활용한 부드러운 슬라이딩 애니메이션 연출을 적용하여 시각적 몰입감을 극대화했으며, 유저별 최고 점수 랭킹 보존 및 데이터 제어를 위한 세부 관리자 대시보드를 통합 제공합니다.

---

### ✨ 주요 기능
#### 1. 유저 (User) 기능
* **회원가입 및 로그인:** `Werkzeug` 비밀번호 해시 암호화를 적용한 안전한 인증 시스템
* **실시간 애니메이션 게임:** 방향키 제어에 따른 타일의 부드러운 이동(Sliding) 및 합치기(Merging) 연출 구현
* **점수 자동 기록:** 게임 오버 시 실시간 최고 타일 및 점수를 서버로 자동 전송
* **글로벌 명예의 전당:** 유저별 역대 최고 스코어 기준 상위 10위 실시간 랭킹 보드 출력

#### 2. 관리자 (Admin) 기능 (CRUD 대시보드)
* **대시보드 조회(R):** 전체 가입 회원 목록 및 역대 게임 점수 기록 데이터 실시간 모니터링
* **회원 권한 수정(U):** 특정 유저의 일반(user) / 관리자(admin) 권한 실시간 스위칭 
* **회원 계정 삭제(D):** 데이터베이스 연결에 기반한 특정 불량 회원 추방 및 계정 영구 삭제
* **조작 점수 제거(D):** 비정상적이거나 악의적으로 등록된 특정 게임 기록을 선택하여 삭제

### 🗄️ DB 구조 (Cloud PostgreSQL)
외부 클라우드 DB 인프라(`Supabase`)를 연동하여 서버 리셋과 관계없이 회원 정보와 스코어를 안전하게 영구 보존합니다.

* **`users` (회원 정보 테이블)**
  * `id` (SERIAL, PK) : 고유 회원 번호
  * `username` (TEXT, UNIQUE) : 유저 아이디
  * `password` (TEXT) : 암호화된 비밀번호 해시값
  * `role` (TEXT) : 회원 권한 (`user` / `admin`)

* **`scores` (게임 기록 테이블)**
  * `id` (SERIAL, PK) : 기록 고유 번호
  * `username` (TEXT) : 플레이어 아이디 (외래 키 역할)
  * `score` (INTEGER) : 최종 획득 점수
  * `max_tile` (INTEGER) : 달성한 최고 블록 숫자 (예: 1024, 2048)
  * `created_at` (TIMESTAMP) : 게임 기록 일시

### 🤖 사용 AI 및 도움받은 내용
* **사용 AI 이름:** Gemini (Google)
* **도움받은 분야 및 내용:**
  * **게임 연출 및 알고리즘 고도화:** 기존의 텍스트 텔레포트 방식 코드에서 타일마다 고유 ID를 부여하고 실시간 격자 위치값을 계산하는 자바스크립트 기반 무빙 애니메이션 알고리즘을 구현하는 데 핵심적인 도움을 받았습니다.
  * **환경 에러 해결 및 DB 마이그레이션:** 로컬 아나콘다 환경에서 생성된 `requirements.txt` 내의 로컬 경로 종속성 에러를 식별 및 정제하고, Render 배포 시 데이터 보존을 위해 SQLite3 코드를 PostgreSQL(`psycopg2`) 규격으로 전환하는 백엔드 DB 설계 및 예외 처리 코드 변환에 기여했습니다.
* **이용하면서 느낀 점:**
  개발 과정에서 막연했던 웹 애니메이션의 구현 원리를 시각적 좌표 개념으로 쉽게 풀어서 설명해 주어 논리적인 구조를 짜는 데 큰 도움이 되었습니다. 단순한 코드 생성을 넘어 배포 환경의 특성(휘발성 파일 시스템)으로 인해 발생할 수 있는 잠재적 크래시 요소를 미리 짚어주고 외부 클라우드 DB 연동이라는 실무적인 아키텍처 대안을 함께 제시해 준 덕분에, 시행착오를 획기적으로 줄이며 완성도 높은 상용 서비스를 완성할 수 있었습니다.
