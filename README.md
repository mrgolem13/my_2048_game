# 2048 웹 게임 프로젝트 (Flask & SQLite3 기반)
92102878 천지인

## 1. 주제 설명
본 프로젝트는 고전 인기 퍼즐 게임인 '2048'을 웹 브라우저 환경에서 즐길 수 있도록 구현한 full-stack 웹 애플리케이션입니다. Flask 프레임워크와 SQLite3 데이터베이스를 연동하여 사용자의 세션을 안전하게 관리하며, 실시간 스코어 기록 저장, 글로벌 랭킹 시스템, 그리고 사용자 및 점수 데이터를 종합적으로 관제할 수 있는 관리자 페이지를 통합 제공합니다.

## 2. 주요 기능
* **사용자 인증 (Auth CRUD):** 세션 기반 회원가입, 로그인, 로그아웃 기능을 지원하며 비밀번호 검증을 통해 개별 사용자 공간을 분리합니다.
* **2048 퍼즐 코어 엔진:** 방향키 입력을 감시하여 블록을 병합하고 실시간 스코어를 계산하는 완전한 가상 보드 매커니즘을 제공합니다.
* **비동기 실시간 랭킹 시스템 (Read):** 게임 세션을 유지한 채 진행도가 초기화되지 않도록 RESTful API(`fetch`) 통신을 활용하여 화면 위에 모달(Modal) 팝업 형식으로 실시간 명예의 전당 Top 10 리스트를 로드합니다.
* **종합 관리자 대시보드 (Admin CRUD):** 관리자 권한(`role='admin'`) 세션이 검증된 계정에게 상시 대시보드 진입을 허용하며, 전체 가입 유저 권한 수정 및 삭제, 비정상 스코어 기록 원격 삭제 기능을 수행합니다.

## 3. DB 구조
데이터베이스는 비임베디드형 경량 파일 RDBMS인 `SQLite3`를 활용하여 2개의 격리된 테이블로 설계되었습니다.

### `users` 테이블 (유저 정보 관리)
* `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT): 유저 고유 식별 번호
* `username` (TEXT): 로그인용 사용자 커스텀 아이디
* `password` (TEXT): 사용자 인증용 비밀번호
* `role` (TEXT, DEFAULT 'user'): 시스템 접근 권한 등급 ('user' / 'admin')

### `scores` 테이블 (점수 및 기록 관리)
* `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT): 기록 고유 식별 번호
* `username` (TEXT): 기록을 달성한 유저의 아이디
* `score` (INTEGER): 최종 게임 종료 및 기록 시점의 누적 점수
* `max_tile` (INTEGER): 해당 게임 세션에서 합성된 가장 높은 블록 수치 (예: 2048)
* `date` (TEXT): 기록이 데이터베이스에 커밋된 날짜 및 시간 데이터

## 4. 느낀 점
프론트엔드와 백엔드를 동시 설계하며 상태 관리의 중요성을 깊이 배웠습니다. 특히 초기 설계 시 랭킹 페이지로 이동할 때마다 HTTP 리다이렉션으로 인해 인메모리 게임 진행 상태가 초기화되던 치명적인 병목이 있었습니다. 이를 해결하기 위해 백엔드에 JSON 전용 라우트를 증설하고 프론트엔드에서 Fetch API 기반의 모달 팝업 구조로 전환하며 비동기 데이터 통신의 강력함을 체감했습니다. 또한 CSS의 박스 모델(Box-model) 계산 착오로 타일들이 좌상단으로 쏠리던 기하학적 버그를 완벽하게 정대칭 수치 계산으로 바로잡으며 완성도 높은 UI 디버깅 경험을 쌓을 수 있었습니다.
